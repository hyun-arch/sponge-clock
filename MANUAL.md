# 🧽 Sponge Clock 사용 메뉴얼

> 긴급 발표용 타이머. **3초 안에 세팅 → 시작**, 그리고 **다른 창을 띄워도 화면 맨 앞에 계속 떠 있는** 타이머.

---

## 1. 개요

Sponge Clock은 발표·세미나·라이브 진행에서 시간을 재기 위한 타이머입니다. 다음 세 가지를 동시에 만족하도록 설계했습니다.

- **빠르다** — 프리셋 클릭 한 번 + `Space`로 즉시 시작
- **항상 보인다** — 미니 창이 모든 창 위에 떠서 발표 슬라이드 위에서도 보임
- **어디서나 된다** — 크롬 확장 / 일반 웹페이지 / 설치형 앱(PWA) 모두 동일하게 동작

---

## 2. 설치 & 실행

### A. 크롬 확장으로 (권장)
1. 주소창에 `chrome://extensions` 입력
2. 우측 상단 **개발자 모드** 켜기
3. **압축해제된 확장 프로그램을 로드** → `sponge clock` 폴더 선택
4. 툴바의 🧽 아이콘 클릭 → **팝업 컨트롤 센터**가 열림
5. 업데이트 후엔 `chrome://extensions`에서 새로고침(↻)

### B. 일반 웹페이지로 (확장 설치 없이)
- `index.html` 또는 `timer.html`을 더블클릭 → 브라우저에서 바로 실행
- 폴더를 정적 호스팅(GitHub Pages·Netlify·Vercel·사내 서버 등)에 올리면 URL로 어디서나 접속

### C. 앱으로 설치 (PWA)
- 호스팅한 주소에서 주소창의 **설치** 아이콘 클릭 → 독립 창 앱으로 설치
- **오프라인에서도 동작** (서비스 워커가 자동 캐시)
- 갱신이 안 보이면 새로고침(`Ctrl+Shift+R`) 한 번

> 설치에 로그인·회원가입은 전혀 필요 없습니다.

---

## 3. 4가지 표시 방식

팝업(툴바 아이콘)에서 원하는 방식을 고릅니다.

| 방식 | 용도 | 특징 |
|------|------|------|
| **팝업** | 잠깐 확인·조작 | 툴바 아이콘 클릭. 프리셋·시작·정지·세 가지 띄우기 버튼 |
| **미니창 (항상 위로)** | 발표 내내 곁눈질 | 화면 **우측 상단 구석**에 작게. 안의 `📌` → 모든 창·앱 위에 고정(PiP) |
| **전체화면** | 무대·강연장 | 링만 거대하게, 컨트롤 숨김 |
| **새 탭** | 큰 화면으로 | 일반 탭에서 크게 보기 |

### 미니창을 "항상 위로" (핵심 기능)
미니창 안의 **📌 버튼**을 누르면 Picture-in-Picture로 전환되어, 발표 슬라이드를 전체화면으로 띄우거나 다른 앱을 써도 구석에서 계속 보입니다. 해제하려면 다시 📌 또는 "항상 위 해제".
- 웹 버전에서도 동일하게 `P` 또는 📌로 현재 페이지를 항상 위로 띄울 수 있습니다.

### 자동 실행
팝업 하단 **"브라우저 시작 시 미니창 자동 실행"** 을 켜두면, 크롬을 켤 때마다 미니 타이머가 자동으로 구석에 떠 있습니다.

## 4. 타이머 모드 (4종)

### ① 카운트다운 (기본)
설정한 시간에서 0까지 줄어듭니다. 0을 지나면 **초과 시간**이 `+MM:SS`로 빨갛게 카운트됩니다(발표가 길어졌을 때 얼마나 넘겼는지 표시).

### ② 스톱워치
0부터 올라갑니다. 링은 1분마다 한 바퀴.

### ③ 다구간 어젠다 타이머
"발표 20분 → Q&A 5분"처럼 구간을 이어서 자동 진행합니다.
- 구간이 끝나면 **자동으로 다음 구간 전환** + 차임 + 음성 안내("Q&A 시작")
- 화면에 **현재 구간 이름 · `1/2`**, 그리고 "다음: Q&A · 전체 25:00 남음" 표시
- **`N` 키 / ⏭ 버튼**으로 즉시 다음 구간 이동
- 마지막 구간을 넘기면 "어젠다 종료"
- 설정 방법: **직접 입력 → "어젠다" 탭** → 구간 추가/삭제, 또는 프리셋(발표+Q&A 등) 선택

### ④ 종료 시각 모드
"오후 3시까지"처럼 절대 시각을 입력하면 그 시각에 0이 되도록 자동 카운트다운합니다(이미 지난 시각이면 내일 기준). 설정 방법: **직접 입력 → "종료 시각" 탭**.

---

## 5. 시간 설정

- **프리셋 칩**: 1·3·5·10·15·20·30·45·60분 클릭 한 번
- **직접 입력**: "직접 입력" 칩 → 대화상자에 3개 탭
  - **시간**: 분:초 직접 입력
  - **종료 시각**: 시:분 (절대 시각)
  - **어젠다**: 여러 구간 + 프리셋
- **+1:00 / `+` 키**: 진행 중에도 1분 추가

---

## 6. 색상 존 & 초과 경고

남은 시간에 따라 링·글로우 색이 자동으로 바뀝니다. 흘끗 봐도 긴급도가 읽힙니다.

| 구간 | 색 | 의미 |
|------|----|----|
| 50% 이상 | 강조색(기본 파랑) | 여유 |
| 50% 이하 | 초록 | 절반 |
| 20% 이하 | 주황 | 경고 |
| 10% 이하 | 빨강 | 위험 |
| 0% 초과 | 빨강 + `+MM:SS` | 시간 초과 |

> 강조색을 바꿔도 **경고(주황)·위험(빨강)은 그대로** 유지되어 긴급도는 항상 동일하게 읽힙니다.

**초과 시 화면 점멸**: 시간을 넘기면 화면 가장자리가 빨갛게 점멸합니다(설정에서 끄기 가능).

---

## 7. 알림

- **사운드**: 1분 전 경고음 · 마지막 10초 틱 · 종료 차임 (Web Audio로 생성, 음원 파일 불필요)
- **음성 안내(TTS)**: "1분 남았습니다 / 30초 남았습니다 / 시간이 종료되었습니다" 음성. 화면을 못 볼 때 귀로 인지. 음성을 켜면 1분·종료 알림은 음성이 대신하고, 10초 카운트는 비프음 유지
- **시스템 알림**: 모든 창이 가려져 있어도 시간이 끝나면 OS 알림으로 알려줌(확장 버전)
- **중복 없음**: 알림은 **보이는 창에서만** 1회 울립니다

## 8. 설정 (⚙ 또는 `S`)

| 항목 | 내용 |
|------|------|
| **테마** | 다크 / 라이트 (밝은 발표장 대응) |
| **강조색** | 블루 · 퍼플 · 그린 · 핑크 · 그래파이트 |
| **음성 안내** | TTS 켜기/끄기 |
| **초과 시 화면 점멸** | 켜기/끄기 |
| **소리** | 🔊 버튼 또는 `M` |
| **자동 실행** | 브라우저 시작 시 미니창 (팝업 하단) |

모든 설정은 **모든 창에 즉시 동기화**되고 다음에도 유지됩니다.

## 9. 단축키 (전체화면·새 탭 화면)

| 동작 | 키 |
|------|----|
| 시작 / 일시정지 | `Space` |
| 리셋 | `R` |
| 전체화면 | `F` |
| 항상 위로 미니 | `P` |
| 설정 | `S` |
| 다음 구간(어젠다) | `N` |
| 1분 추가 | `+` |
| 소리 끄기/켜기 | `M` |

## 10. 발표 시나리오별 추천

- **5분 번개 발표**: 팝업에서 `5` → `Space`. 또는 미니창 띄우고 📌.
- **무대 키노트**: 전체화면(`F`) → 프리셋 → `Space`. 음성 안내 켜두면 화면 안 봐도 됨.
- **발표 + Q&A**: 어젠다 탭에서 "발표20+Q&A5" 프리셋 → 시작. 구간 전환 자동.
- **세션 종료 시각 고정**: 종료 시각 탭에 끝나는 시각 입력.
- **딴 작업하며 라이브 진행**: 미니창 📌(항상 위) → 다른 앱 위에 떠 있음.

## 11. 구조 (개발자용)

- `engine.js` — 공유 타이머 상태 머신. 확장이면 `chrome.storage`, 웹이면 `localStorage` 자동 전환. 타임스탬프 기반이라 모든 창이 정확히 동기화
- `background.js` — 서비스 워커: 창 열기 / 알람 / 자동 실행
- `popup.*` — 팝업 컨트롤 센터
- `mini.*` — 미니 플로팅 + 항상위(PiP)
- `timer.*` — 전체화면 / 새 탭 발표 화면 + 설정·어젠다 대화상자
- `sound.js` / `voice.js` / `cues.js` — 사운드·음성 알림
- `sw.js` / `manifest.webmanifest` — PWA(오프라인·설치)

## 12. 문제 해결

- **새 기능이 안 보여요** → 확장: `chrome://extensions` 새로고침(↻). 웹/PWA: `Ctrl+Shift+R`.
- **항상 위(📌)가 안 돼요** → Document Picture-in-Picture 미지원 브라우저일 수 있음. 최신 Chrome/Edge에서, `https` 또는 `localhost`에서 가장 잘 동작.
- **소리가 안 나요** → 브라우저 정책상 첫 사용자 클릭 후 활성화됨. 한 번 클릭/시작하면 이후 정상.
- **음성이 안 나와요** → 설정에서 음성 안내 ON 확인. OS에 한국어 음성이 있어야 함.
